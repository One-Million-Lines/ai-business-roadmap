Build a frontend-only web app in React using shadcn/ui, Tailwind CSS, and TypeScript.

Goal:
Create a visually clear, professional, roadmap.sh-inspired interface for an “AI Integration Roadmap for Businesses”.
This is not a generic mind map. It is a structured, zoomable, browsable knowledge and decision map that helps business and technical users understand how AI can be introduced into existing company workflows.

Important:
- No backend
- No authentication
- No database
- No API calls
- Use static mock data inside the app
- The app must feel polished, modern, easy to scan, and highly useful visually
- The UI should prioritize clarity over decoration
- The result should be easy to extend later with real data

Tech requirements:
- React
- TypeScript
- Tailwind CSS
- shadcn/ui components
- Use lucide-react for icons
- Use Framer Motion for subtle animations
- Prefer a clean modular component structure
- Build everything in one frontend app
- No external graph libraries unless truly necessary; prefer custom layout with standard React + CSS
- Make code production-style, readable, and componentized

Core product idea:
The page represents a structured roadmap similar in spirit to roadmap.sh, but focused on business AI integration.
It should visually explain:
1. business problems
2. use cases
3. architecture
4. integrations
5. memory/context
6. execution patterns
7. trust/governance
8. measurement
9. rollout

The roadmap should support both:
- waterfall-style understanding from top to bottom
- parallel tracks such as business, technical, AI capability, governance

Design philosophy:
- Users must understand the structure within seconds
- The app should feel like a strategic map, not a random canvas
- It should be useful for consultants, founders, product people, technical architects, and operations leaders
- It must feel more like a “decision map” than a learning article
- Every visual choice should improve comprehension
- Dense information is okay, but must remain highly navigable

Overall layout:
Create a single-page application with the following major zones:

1. Top header
- Product title: “AI Integration Roadmap”
- Small subtitle explaining that this is a strategic map for introducing AI into business workflows
- Search input
- View mode toggle
- Legend button
- Filters button

2. Main roadmap canvas area
- The core of the page
- Large scrollable and responsive layout
- Structured into horizontal sections or vertical layers
- Must visually show both:
  - layers (Problem, Use Case, Architecture, etc.)
  - tracks (Business, Technical, AI, Governance)
- Use cards/nodes arranged in a grid or flow-based layout
- Use arrows, connectors, or subtle visual linking where helpful
- Should resemble roadmap.sh in clarity and flow, but can be more card-based and enterprise-friendly

3. Right-side detail panel or modal drawer
- Opens when clicking a node
- Shows rich details about the selected item
- Should feel like a knowledge inspector
- Easy to scan
- Includes sections like description, why it matters, dependencies, risks, metrics, examples, maturity model

4. Optional compact overview/minimap panel
- Shows the big picture
- Helps users understand where they are in the roadmap
- Can be simplified if needed, but include at least some summary aid for orientation

1. Layered View

- Rows = layers
- Columns = tracks
- Nodes appear where they belong
- Easy to scan top-to-bottom and left-to-right

2. Flow View

- Better for understanding progression from problem to rollout
- Show clearer connectors between nodes

3. Compact View
- Denser view for experienced users
- Smaller cards
- Better for browsing more nodes quickly

Behavior requirements:
- Clicking a node opens details in side panel or drawer
- Hovering a node should subtly highlight it and optionally its dependencies
- Selecting a node should visually emphasize related nodes
- Search should filter in real time
- Filters should combine cleanly
- Empty states should be friendly and useful
- Transitions should be smooth but subtle
- The app must remain understandable on first use

Visual behavior:
- Keep the interface clean and calm
- Use spacing, grouping, and typography to make complexity digestible
- Avoid visual clutter
- Use cards with rounded corners, light shadows, and strong hierarchy
- Use color sparingly and meaningfully
- Important nodes should stand out without being flashy
- Connections should help understanding, not create noise

Accessibility and UX:
- Keyboard-friendly where practical
- Good contrast
- Responsive for desktop first, but usable on tablet
- On mobile, collapse roadmap into stacked sections rather than forcing a giant unusable canvas
- Side panel should become a bottom drawer or full-screen sheet on smaller screens
- Text should remain readable at all times
- No tiny labels that require zooming

Styling guidance:
- Modern SaaS / strategy tool feel
- Professional, minimal, clear
- More like a premium internal planning tool than a playful mind map
- Use typography hierarchy carefully
- Section titles should be prominent
- Nodes should be easy to differentiate without overwhelming color use
- Favor soft borders, muted backgrounds, and clean spacing
- Use sticky headers where helpful
- Use subtle animations for hover, panel open, filter changes, and selected state

Important implementation guidance:
- Keep data separate from UI
- Use a dedicated mock data file
- Use derived selectors/helpers for filtering and grouping
- Avoid hardcoding layout too much
- Make it easy to replace mock data later
- Keep code modular and maintainable
- Add clear inline comments only where useful
- Avoid overengineering

Layout logic:
For the Layered View:
- Render sections by layer
- Within each layer, group by track
- Nodes can span multiple tracks if needed, but keep display simple
- If a node belongs to multiple tracks, visually indicate it with secondary badges or subtle multi-track markers

For the Flow View:
- Show selected or featured paths
- Example path:
  Problem → Use Case → Architecture → Integration → Execution → Trust → Measurement → Rollout
- Keep it readable and not overly diagrammatic

Initial default state:
- Start in Layered View
- Show all layers
- Show 1 selected “featured path” near the top as an example
- Preload with one highlighted use case, such as “Auto-follow-up on inbound leads”

Usefulness requirements:
The app should teach by structure.
A user should be able to answer questions like:
- Where do I start with AI in my business?
- What comes after identifying a use case?
- What technical components are needed?
- Where do governance and approval fit?
- How do I measure ROI?
- How does a use case mature from manual to autonomous?

Do not build a generic dashboard.
Do not build a generic kanban board.
Do not build a random network graph with messy lines everywhere.
Build a structured, strategic roadmap product.
